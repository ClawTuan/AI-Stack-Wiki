## 一句话总结

Attention Residuals (AttnRes) 提出用深度维度上的注意力机制替代固定权重的残差连接，使每层能选择性地聚合之前层的输出，解决了PreNorm架构中隐藏状态随深度增长导致的信息稀释问题。

## 核心信息

- 机构：Kimi Team
- 发布时间：2026年3月
- 核心创新：将固定权重的残差连接替换为可学习的深度注意力机制
- 实现变体：
  - Full AttnRes：每层关注所有之前层的输出
  - Block AttnRes：将层分组为块，关注块级表示以减少内存开销
- 应用场景：大型语言模型(LLM)训练与推理

## L0：核心概念与动机

### 残差连接的局限性

传统残差连接使用固定的单位权重累加所有层输出，存在以下问题：
- 隐藏状态随深度线性增长(O(L))
- 每层贡献被逐渐稀释
- 无法选择性地访问早期层信息
- 深度梯度分布不均匀

### Attention Residuals的核心思想

借鉴Transformer在序列维度的注意力机制，将其应用于深度维度：
- 每层使用可学习的伪查询向量(w_l)对之前所有层的输出进行注意力加权
- 注意力权重(α_i→l)通过softmax计算，实现内容感知的选择性聚合
- 保持与标准残差连接相同的梯度流特性

## L1：技术细节

### 数学形式

标准残差连接：
```
h_l = h_{l-1} + f_{l-1}(h_{l-1})
```

Attention Residuals：
```
h_l = um_{i=0}^{l-1} α_{i→l} · v_i
```
其中：
- α_{i→l} 是注意力权重，满足 um_{i=0}^{l-1} α_{i→l} = 1
- v_i 是第i层的输出表示
- α_{i→l} 通过softmax(q_l·k_i / √d)计算，q_l是可学习的查询向量

### Block AttnRes优化

为解决大规模训练的内存和通信开销：
1. 将L层分为N个块，每个块包含S=L/N层
2. 块内使用标准残差连接累加层输出
3. 块间使用注意力机制聚合块级表示
4. 内存和通信开销从O(Ld)降低到O(Nd)

### 系统优化

- **交叉阶段缓存**：在流水线并行训练中缓存块表示，减少冗余通信
- **两阶段计算策略**：
  - 阶段1：并行计算块内所有层的块间注意力
  - 阶段2：顺序计算块内注意力并通过在线softmax合并结果
- **内存高效预填充**：在推理时沿序列维度分片块表示，减少内存占用

## L2：实验结果与分析

### 扩展规律

在5个模型规模上的实验显示：
- Block AttnRes与Full AttnRes在所有规模上均优于基线
- Block AttnRes可达到基线模型1.25倍计算量的性能
- 模型规模越大，Full AttnRes与Block AttnRes的差距越小

### 训练动态分析

- **验证损失**：AttnRes全程保持较低损失，衰减阶段差距扩大
- **输出幅度**：AttnRes控制了输出幅度随深度的增长，形成有界的周期性模式
- **梯度分布**：AttnRes使各层梯度分布更加均匀，解决了早期层梯度过大的问题

### 下游性能

在48B参数的Kimi Linear模型上，Block AttnRes在所有基准测试中表现更优：
- 多步推理任务（如GPQA-Diamond）提升+7.5
- 数学任务（如Math）提升+3.6
- 代码生成（如HumanEval）提升+3.1
- 知识型任务（如MMLU）提升+1.1

## 与现有方法对比

| 方法 | 特点 | 性能 |
|------|------|------|
| PreNorm基线 | 固定权重累加 | 基准 |
| DenseFormer | 固定标量系数 | 无提升 |
| mHC | 多流混合矩阵 | 中等提升 |
| Full AttnRes | 全层注意力 | 最佳性能 |
| Block AttnRes | 块级注意力 | 接近Full性能，内存友好 |

## 应用前景

- 可作为现有Transformer架构的即插即用替换组件
- 训练开销<4%，推理延迟开销<2%
- 特别适合需要深度信息交互的复杂任务
