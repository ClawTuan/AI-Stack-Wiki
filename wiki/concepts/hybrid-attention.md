## L0：概念定义

混合注意力架构（Hybrid Attention）是一种针对长上下文场景优化的注意力机制设计，通过组合多种注意力变体（如压缩稀疏注意力、重度压缩注意力、滑动窗口注意力等），在不同层级和场景下使用最适合的注意力策略，从而在保持模型能力的同时大幅降低长序列的计算和内存开销。

**核心思想**：没有一种注意力机制适合所有场景，通过智能组合多种机制，实现"该稀疏的地方稀疏，该密集的地方密集"。

## L1：核心组件与工作原理

### 为什么需要混合注意力？

标准Transformer的注意力机制面临的核心挑战：

1. **二次复杂度**：注意力计算随序列长度呈O(n²)增长
2. **KV缓存爆炸**：长序列的KV缓存占用大量内存
3. **长距离依赖**：全局注意力在长序列中效率低下
4. **局部与全局的平衡**：不同任务对局部细节和全局概览的需求不同

### 混合注意力的典型组成

#### 1. 压缩稀疏注意力（Compressed Sparse Attention, CSA）

**核心思想**：先压缩KV缓存，再在压缩后的表示上执行稀疏注意力。

**工作流程**：
1. **KV压缩**：沿序列维度将每m个Token的KV表示压缩为1个条目
2. **索引器选择**：使用轻量级索引器选择最相关的Top-k个压缩KV条目
3. **稀疏注意力**：仅在选中的KV条目上计算注意力

**关键参数**：
- 压缩率m：通常为4（4:1压缩）
- Top-k：选择512或1024个最相关条目
- 索引器：独立的查询头，负责选择相关KV

**适用场景**：需要全局信息但可接受近似的长序列任务

#### 2. 重度压缩注意力（Heavily Compressed Attention, HCA）

**核心思想**：使用更激进的压缩率，在高度压缩的KV上执行密集注意力。

**工作流程**：
1. **重度压缩**：沿序列维度将每m'个Token的KV表示压缩为1个条目（m'通常为128）
2. **密集注意力**：在压缩后的KV上执行标准的全注意力

**关键参数**：
- 压缩率m'：通常为128（128:1压缩）
- 保持密集：虽然KV被压缩，但注意力是密集的

**适用场景**：需要全局概览且对细节不敏感的任务，或作为模型的早期层

#### 3. 滑动窗口注意力（Sliding Window Attention, SWA）

**核心思想**：每个Token只关注其附近的局部上下文。

**工作流程**：
1. **窗口定义**：定义固定大小的局部窗口（如128个Token）
2. **局部注意力**：每个查询只与窗口内的Key计算注意力

**关键参数**：
- 窗口大小n_win：通常为128或256

**适用场景**：需要精细局部感知的任务，或作为辅助分支增强局部能力

### 混合策略设计

#### 层级分配

典型的混合策略：
- **早期层（1-2层）**：使用HCA或SWA，建立初步表示
- **中间层**：CSA和HCA交错使用
- **所有层**：SWA作为额外分支，增强局部感知

#### 动态选择

更高级的设计可根据输入动态选择注意力类型：
- 短序列：使用标准注意力
- 中等长度：使用CSA
- 超长序列：使用HCA或CSA+HCA组合

## L2：技术细节与实现

### CSA的详细实现

#### 压缩过程

对于每m个连续的Token，其KV表示被压缩：

```
KV_compressed[i] = Compress(KV[m*i : m*(i+1)])
```

压缩函数可以是：
- 平均池化
- 加权平均（基于位置或重要性）
- 可学习的压缩网络

#### 索引器设计

索引器是一个独立的注意力机制：
- 使用独立的查询头（如64个头，维度128）
- 在压缩后的KV上计算相关性分数
- 选择Top-k个最相关的压缩KV条目

#### 稀疏注意力

在选中的Top-k个KV条目上计算标准注意力：

```
Attention(Q, K_selected, V_selected) = softmax(Q·K_selected^T / √d)·V_selected
```

### HCA的详细实现

#### 重度压缩

压缩率m'通常远大于CSA的m（如128 vs 4）：

```
KV_heavily_compressed[i] = Compress(KV[m'*i : m'*(i+1)])
```

#### 密集注意力

在高度压缩的KV上执行标准注意力，计算复杂度为O(n·n/m')，其中n/m'是压缩后的序列长度。

### KV缓存管理

混合注意力引入了异构KV缓存的挑战：

#### 缓存类型

1. **CSA缓存**：压缩后的KV条目，大小为原始序列的1/m
2. **HCA缓存**：重度压缩的KV条目，大小为原始序列的1/m'
3. **SWA缓存**：完整的局部窗口KV条目
4. **未压缩尾部**：等待压缩的未完整块

#### 管理策略

- **状态缓存**：为SWA和未压缩尾部预分配固定大小的缓存池
- **分页管理**：对CSA和HCA缓存使用类似PagedAttention的管理
- **对齐要求**：缓存块大小需为lcm(m, m')的倍数，以满足不同层的对齐需求

### 效率分析

#### 计算复杂度对比

| 注意力类型 | 计算复杂度 | 内存复杂度 |
|-----------|-----------|-----------|
| 标准注意力 | O(n²·d) | O(n·d) |
| CSA | O(n·k·d) + O(n·n/m·d_index) | O(n/m·d) |
| HCA | O(n·n/m'·d) | O(n/m'·d) |
| SWA | O(n·w·d) | O(w·d) |

其中：
- n：序列长度
- d：隐藏维度
- k：Top-k选择数
- m：CSA压缩率
- m'：HCA压缩率
- w：窗口大小

#### 实际效率提升

以DeepSeek-V4为例，在1M Token场景下：
- CSA（m=4）+ HCA（m'=128）组合
- 相比标准注意力，FLOPs降低约3.7-9.8倍
- KV缓存降低约9.5-13.7倍

## 与其他技术的关系

### 与标准注意力的对比
- **标准注意力**：全局密集，能力最强但效率最低
- **混合注意力**：智能组合，在能力和效率间取得平衡

### 与纯稀疏注意力的对比
- **纯稀疏注意力**（如Sparse Transformer、Longformer）：固定稀疏模式
- **混合注意力**：动态选择，更灵活地适应不同需求

### 与线性注意力的对比
- **线性注意力**（如Linear Attention、Performer）：通过核技巧降低复杂度
- **混合注意力**：保持标准注意力的表达能力，通过压缩和选择降低开销

## 发展趋势

1. **自适应混合**：根据输入动态调整注意力组合
2. **学习压缩**：使用可学习网络替代固定压缩函数
3. **多级压缩**：更多压缩级别（如2:1、4:1、16:1、128:1）
4. **硬件协同设计**：与专用硬件（如稀疏注意力加速器）协同优化
5. **跨模态扩展**：将混合注意力应用于视觉、音频等多模态场景

## 代表工作

- **DeepSeek-V4**：CSA + HCA + SWA的混合架构，支持1M Token上下文
- **Longformer**：局部窗口 + 全局稀疏注意力的早期探索
- **BigBird**：随机 + 窗口 + 全局注意力的组合
- **Sparse Transformer**：固定稀疏模式的注意力
